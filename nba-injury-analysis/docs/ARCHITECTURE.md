# Architecture

## Goal

Estimate which basketball movement mechanics raise injury risk, by comparing the
mechanics present in injuries against how often those same mechanics occur in
normal play.

## Three phases

```
Phase A  injury clips ─▶ ClipMetricLabel  (manual ground truth, CV-assisted)
Phase B  all plays    ─▶ PlayRecord       (base-rate exposures, with n_exposures)
Phase C  A + B        ─▶ RiskEstimate      (relative risk / adjusted regression)
```

The three phases are independent and share only the **schema layer**
(`src/nba_injury/schema/`). They can be built, run, and tested separately.

### Phase A — injury metrics (WORKING prototype)

Pipeline: `clip → pose estimation (tracked) → derived signals → JSON`.

- `io/video.py` reads frames (decord, OpenCV fallback).
- `pipelines/phase_a_injury/pose_estimation.py` wraps **Ultralytics YOLO-pose**
  with tracking behind the `PoseEstimator` protocol, so MediaPipe / MMPose can
  be swapped in later. Outputs `ClipPoseSeries`.
- `select_subject_track()` picks which player to analyze (most persistent +
  central track), with a `--track-id` override. **This is not injured-player
  identification** — a documented limitation; the human labeler confirms.
- `derived_signals.py` (pure, deterministic) computes the airborne phase,
  takeoff/landing frames, a low-confidence landing-feet *suggestion*, and a
  motion-state *suggestion*.
- `run.py` is the CLI; it writes `<clip>.parquet`, `<clip>.meta.json`, and
  `<clip>.signals.json`.

**Auto-detectable vs manual** (key honesty point):

| Metric | Status |
|---|---|
| airborne phase, takeoff/landing frame | auto (computed) |
| landing-feet count (1 vs 2) | auto *suggestion*, low confidence |
| motion state (moving/stationary) | auto *suggestion* |
| takeoff-foot count (1 vs 2) | **manual** |
| non-feet landing surface | **manual** |
| interference (stepped-on foot / shove) | **manual** |

The pose signals are **assistance**, fed into the labeling tool. Manual labels
are ground truth.

### Phase B — base rates (STUB)

`pipelines/phase_b_baserates/run.py` will scan plays over a chosen time frame
and emit one `PlayRecord` per qualifying action, with the **same metric enums**
as `ClipMetricLabel` plus `n_exposures` (the Phase C denominator).
`summarize_exposures()` already aggregates counts per metric value.

### Phase C — cross-reference (STUB)

`exposure.py` builds 2×2 `ExposureTable`s (exposed vs reference, injuries vs
exposures). `models.py` will compute:

1. **Relative risk / incidence-rate ratio** per mechanic, with CIs
   (scipy/statsmodels) — primary, interpretable.
2. **Multivariable regression** (logistic, or Poisson/neg-binomial with a
   `log(n_exposures)` offset) to adjust for confounders (player, event type).

Inference-first, not balanced-accuracy prediction, because injuries are rare.

## Data flow & storage

- Records (`InjuryEvent`, `ClipMetricLabel`, `PlayRecord`) ↔ JSON (pydantic).
- Pose series ↔ long-format **parquet** (one row per frame×track×keypoint) +
  small JSON metadata sidecar (`io/storage.py`).
- All clips and derived data live under `data/` and are **gitignored**.

## Key design decisions & risk mitigations

| Risk | Mitigation |
|---|---|
| Footage copyright | `data/` gitignored; only numeric derivatives kept; see LEGAL_LICENSING.md |
| Broadcast pose inaccuracy / occlusion | pluggable `PoseEstimator`; per-keypoint confidence stored; CV is assistive only |
| Rare injuries (class imbalance) | rate-ratio / exposure-offset inference, not balanced classifiers |
| Exposure denominator ambiguity | dedicated `exposure.py`; identical metric enums in labels and plays |
| Which of 10 players is injured | manual `--track-id` override now; tracking heuristic; human confirms |
| Taxonomy reliability | two labelers + adjudication; compute Cohen's kappa early |

## Environment notes

- `ffmpeg` (system) normalizes clip fps; optional but recommended.
- First pose run downloads YOLO-pose weights over the network.
- A synthetic non-NBA fixture clip is generated by `scripts/verify_scaffold.py`
  for CI.
